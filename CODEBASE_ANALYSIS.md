# SevaSync Codebase Analysis Report

## Executive Summary

SevaSync is a disaster volunteer coordination platform with two frontend applications (PWA for volunteers, Dashboard for coordinators) and a backend API. The codebase shows a well-structured approach with PWA-first design principles, offline-first architecture, and organized state management.

---

### Current State: PARTIALLY IMPLEMENTED
Location: `frontend-pwa/`

#### What Exists:

  - `useDashboardAnalytics()` - Dashboard statistics

#### What's Missing:

   - No `QueryClientProvider` wrapper found in App.tsx
   - No explicit QueryClient initialization

2. **No PWA Integration**
   - PWA (frontend-pwa) doesn't use React Query
   - Uses Zustand stores instead with manual fetch

3. **No Query Synchronization**
   - No integration between React Query cache and offline sync queue
   - Dashboard and PWA have different caching strategies

4. **Missing Features:**
   - No ReactQueryDevtools
   - No query persistence to IndexedDB
   - No background refetch configuration
   - No custom retry strategies

## 3. OFFLINE & CACHING PATTERNS

### Current State: COMPREHENSIVE IN PWA, BASIC IN DASHBOARD

#### PWA Offline Architecture:

**IndexedDB Schema** (`frontend-pwa/src/lib/db.ts`)
```
Database: 'sevasync-offline' (v1)

Stores:
1. tasks
   - Key: id (string)
   - Indexes: by-status, by-sync
   - Fields: id, title, description, urgency, status, location, 
             estimatedHours, disasterName, assignedAt, syncStatus, updatedAt

2. syncQueue
   - Key: id (autoincrement)
   - Fields: action, payload, createdAt, retries
   - Actions: accept_task, complete_task, update_location, wellness_checkin

3. volunteer
   - Key: id
   - Fields: id, name, phone, skills, token, currentLat, currentLon, lastSyncAt
```

**Sync Queue System** (`frontend-pwa/src/lib/offline-sync.ts`)
- Exponential backoff retry logic: 1s, 2s, 4s, 8s, 16s, 32s (max 60s)
- Jitter: ±25% randomness
- Max retries: 5 (configurable)
- Processes queue on connectivity recovery

**Offline State Management** (`frontend-pwa/src/stores/offlineStore.ts`)
```typescript
- isOffline: boolean
- syncQueue: SyncQueueItem[]
- pendingSyncCount: number
- lastSyncTime: Date | null
- syncInProgress: boolean
```

**Network Detection** (`frontend-pwa/src/hooks/useOffline.ts`)
- Listens to `window.online/offline` events
- Updates offline state in real-time
- Exposed via `useOffline()` hook

**API Fallback Pattern** (`frontend-pwa/src/lib/api.ts`)
```typescript
If network request fails:
  - Check if error is 'offline'
  - If offline: Queue action for later sync
  - If error: Return error to caller
  - On recovery: Process queue automatically
```

#### Dashboard Caching:

**localStorage Strategy** 
- Token storage: `token` key
- Draft persistence:
  - Task creation draft: `taskCreationDraft`
  - Disaster activation draft: `disasterActivationDraft`
- Manual JSON serialization

**Vite PWA Plugin** (Runtime Caching)
- API cache: NetworkFirst strategy
- 24-hour TTL, 100 entry limit
- 10-second network timeout

#### What's Missing:

1. **Dashboard Offline Support**
   - No IndexedDB in dashboard
   - No sync queue
   - No offline indicators
   
2. **Sync Conflict Resolution**
   - `syncStatus: 'conflict'` defined but not handled
   - No merge strategy for concurrent edits
   
3. **Progressive Data Loading**
   - No stale-while-revalidate pattern
   - No optimistic updates
   
4. **Background Sync API**
   - Not implemented despite being ideal for PWA
   - Currently manual queue processing

## 4. TASK MANAGEMENT DATA STRUCTURES

### Backend Schema (Prisma Models):

**Task Model Fields:**
```
- id: UUID (primary)
- disasterId: UUID (foreign key)
- title: string (required, 5-200 chars)
- description: string (optional, max 2000)
- status: enum (OPEN, ASSIGNED, IN_PROGRESS, COMPLETED, CANCELLED)
- urgency: enum (LOW, MEDIUM, HIGH, CRITICAL)
- type: enum (TaskType)
- latitude: number (-90 to 90)
- longitude: number (-180 to 180)
- estimatedHours: integer (1-100, optional)
- requiredSkills: string array (max 10)

- completedAt: timestamp (optional)
```
