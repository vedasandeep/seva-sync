## Project Summary

SevaSync is now a **resilient offline-first Progressive Web App (PWA)** that enables volunteers to work in low-connectivity disaster zones without losing progress. The app maintains full functionality offline, queues updates locally, syncs automatically when online, and detects/resolves conflicts intelligently.

### Phase 9: Backend Sync APIs ✅
**3 New Endpoints:**

1. **POST /api/v1/sync/bulk**
   - Accepts array of sync items from volunteer app
   - Detects conflicts via version/lastUpdatedAt/updatedBy
   - Returns: { syncedItems, failedItems, conflictItems }
   - Volunteer auth required

2. **GET /api/v1/sync/status**
   - Returns sync health status
   - Includes timestamp and volunteer ID
   - For monitoring sync health from dashboard

3. **POST /api/v1/sync/conflict-resolution**
   - Resolves conflicts by accepting user's choice
   - Applies local or server version based on resolution
   - Increments version on update
   - Updates updatedBy and updatedAt fields

## Architecture Overview

### Frontend PWA Stack
- **Framework**: React + TypeScript
- **Build**: Vite with vite-plugin-pwa
- **State**: Zustand (offlineStore)
- **Database**: IndexedDB (via dexie-like patterns)
- **Styling**: Tailwind CSS
- **Icons**: Lucide React

### Backend API Stack
- **Framework**: Express.js + TypeScript
- **Database**: PostgreSQL with Prisma ORM
- **Auth**: JWT (volunteer tokens)
- **Sync Module**: New sync.service.ts with conflict detection

### Data Flow
```
Volunteer App (Offline)
    ↓
Accept/Complete Task → Queued in IndexedDB
    ↓
Background Sync (every 60s)
    ↓
POST /api/v1/sync/bulk
    ↓
Backend detects conflicts via version/timestamp
    ↓
If conflict: Queue in conflicts list → Show modal
If synced: Mark as synced, update version
If failed: Retry with backoff
    ↓
Sync Queue Drawer shows status in real-time
```
