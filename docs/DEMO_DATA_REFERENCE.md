## Key Features Demonstrated

### Geospatial Data
- All volunteers have realistic GPS coordinates within ±0.1° of their region center
- All tasks have GPS coordinates within ±0.05° of disaster epicenter
- Enables testing of "nearby" filtering and distance calculations

### Temporal Data
- Tasks created within last 5 days
- Assignments and completions show realistic time progression
- Activity levels vary realistically (fresh vs. burnt-out volunteers)

### Relationships
- Tasks are linked to disasters and assigned volunteers
- Task logs track assignment and completion history
- Wellness check-ins show volunteer state over time
- IVR logs show interaction history

### Edge Cases Included
- Some volunteers with zero tasks completed
- Mix of small and large task assignments
- High burnout volunteers alongside fresh ones
- Offline volunteers mixed with active ones
- Tasks with multiple urgency levels

## Using Demo Data in Tests

### Reset Demo Data
```bash
npm run demo:reset
```
Clears all tables and re-seeds. Expected output:
```
Created 3 users
Created 3 disasters
Created 50 volunteers
Created 100 tasks
Created ~45 task logs
Created ~30 wellness check-ins
Created ~40+ IVR logs
```

### Backup Before Large Tests
```bash
npm run db:backup
```
Creates timestamped SQL file in `backend/backups/`

### Restore from Backup
```bash
npm run db:restore backups/backup-2026-04-22-14-30-45.sql
```

## Testing Scenarios with Demo Data

### 1. Volunteer Skill Matching
- Pick a RESCUE task from earthquake (requires rescue skill)
- See matching volunteers: 10+ in Shimla region with rescue skill
- Test proximity filtering: Closer volunteers rank higher

### 2. Burnout Detection
- Filter volunteers with burnoutScore > 75
- Dashboard shows "High Risk" badge
- Auto-assign excludes these volunteers

### 3. Offline Sync Testing
- Use PWA as volunteer with lastActiveAt > 24 hours ago
- Simulate offline: Disable network
- Log tasks locally
- Re-enable network: Tasks sync to backend

### 4. Language Preferences
- Filter volunteers by language (hi/en/te/as)
- IVR responses use correct language
- Task descriptions use volunteer's preferred language

### 5. Task Status Flow
- OPEN → IN_PROGRESS (volunteer accepts)
- IN_PROGRESS → COMPLETED (volunteer completes)
- View task logs showing time progression

### 6. Disaster Overview
- Dashboard shows 3 active disasters with maps
- See task distribution (earthquake: 35, flood: 40, wildfire: 25)
- See volunteer assignments by disaster

### 7. Real-time Updates
- Assign task to volunteer via dashboard
- PWA receives notification in real-time (Socket.io)
- Task appears in volunteer's task list

### 8. IVR Integration
- Test IVR endpoints with sample call data
- Verify correct volunteer lookup by phone
- Test multi-language responses
